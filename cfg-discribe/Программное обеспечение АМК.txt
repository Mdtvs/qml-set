	
Программное обеспечение АМК. ПО АРМ "Метеоролога" версия 1.4.8 от 12.2010, Кофиг АМК текущая версия 83.2.2.
 
Заголовок:
Обновление ПО.
Исправлены ошибки обнаруженные в процессе эксплуатации предыдущей версии АРМ. Актуальная версия 1.4.8
Актуальная версия конфигурационног файла 83.2.2
 
Описание проблемы:
Программное обеспечение АМК
Не правильно формируется телеграмма.
Не выводятся группы в телеграмме.
Неотображается давление в АРМ
 
Решение:
В документе содержит

  1. Установочный файл АРМ "Метеоролога" 1.4.8 от 12.2010 (работает только с версией конфига 83.2.0 и выше)
  2. Конфигурационный файл KN01v83 (работает с АРМ V1.4.8)
  3. Документация на АРМ. Руководство Администратора. Руководство пользователя. Настройки прав пользователя для синхронизации часов компьютера.

Изменения в версии (более подробное описание изменений в файле "version AMK.txt").

2. Добавлены группы.
В параметрах AgreementХ первая (единственная) цифра означает 0 - отключена или 1 - включена.
 
  3 Раздел.
      Группа 4E'sss. Перенесена из раздела 5. За формирование группы отвечает параметр Agreement54.
      Группа 911ff. Максимальная скорость ветра более 18 м/с за 3-х часовой интервал. Группа формируется и передается автоматически. Группа 910ff упразднена за не надобностью, т.к. скорость выбирается автоматически из всего интервала.
  Раздел 5.
      Группа  5snT24T24T24. Среднесуточная температура воздуха. За формирование группы отвечает параметр Agreement55.
      Пример:
              spset Agreement55 "103"
              Группа 5snT24T24T24 передается в срок 03 часов UTC (ВСВ)
      Группа 530f12f12. Максимальная скорость ветра за полусутки. За формирование группы отвечает параметр Agreement553.
      Пример:
              spset Agreement553 11200
              Группа 530f12f12 передается в сроки 12 и 00 UTC (ВСВ)
      Группа 912ff. Максимальна скорость ветра за день (сутки). За формирование группы отвечает параметр Agreement59.
      Пример:
              spset Agreement59 "103"
              В примере показана что группа передается в срок 03 UTC (ВСВ)

3. Параметры с которыми связано большинство ошибок.
     
      Раздел spset.
          FTP_block - в секундах задается время для отправки сообщений на локальный FTP сервер. Рекомендуемое значение для ПК№1 86400, для ПК№2 0
          KN01_StartTime - начало формирования телеграммы КН-01. Должно быт 3004.
          Настройки передачи групп.
            spset Agreement31 1
            spset Agreement52 1
            spset Agreement53 0
            spset Agreement54 0
            spset Agreement57 0
            spset Agreement56 1
            spset Agreement55 "103"
            spset Agreement553 "10315"
            spset Agreement59 "103"
          Veget - начало и окончание вегетативного периода. spset Veget "01:01-31:12"
          KN01EditTime - время в секундах на редактирования визуально наблюдаемой информации должно быть в диапазоне от 600 до 1200

ПРИМЕЧАНИЯ
Добавлена опция сохранения сформированных телеграмм КН-01 и суточных архивов станций на локальном компьютере. Для этого необходимо установить на компьютере наблюдателя локальный FTP сервер, например File Zilla, настроить его в соответствие с инструкцией (см. Настройка FTP сервера.doc).
Скачать FTP сервер можно по этой ссылке.
В следующей верси АРМа "Метеоролога" планируется создать функцию восстановления данных по архивам станции для этого необходима организация локального FTP сервера.
На компьютере ПК№2 может потребоваться отключения функции отправления данных на локальный FTP сервер. Для этого задайте значение параметра spset FTP_block 0. Для всех остальных станций оно должно быт равно 86400.

ВНИМАНИЕ! Используйте тестовые версии конфигов на свой страх и риск. Если у вас нет достаточно знаний по работе с контроллером рекомендуем дождаться выпуска финальной версии.
ВНИМАНИЕ! Ни когда не устанавливайте тестовые конфигурационные файлы на удаленных и труднодоступных станциях.
